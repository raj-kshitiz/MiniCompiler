# Minimal Arithmetic Compiler

A lightweight, minimal compiler and virtual machine for evaluating arithmetic expressions, written entirely in C. 

This project demonstrates the complete pipeline of a traditional compiler: it takes a source string, tokenizes it, builds an Abstract Syntax Tree (AST), generates stack-machine Intermediate Representation (IR), and executes it on a custom Virtual Machine.

## Features

* **Interactive REPL:** Type expressions directly into the terminal and get instant results. Fully compatible with integrated IDE terminals (like VS Code).
* **File Execution:** Read and evaluate math scripts from text files.
* **Pipeline Visibility:** Verbose mode prints out the exact Tokens, an ASCII AST tree, and the generated stack-machine instructions.
* **Modular Architecture:** Cleanly separated into standard C headers and source files for easy maintenance.
* **Built-in Test Suite:** Instantly verify the compiler's accuracy with a suite of automated unit tests.
* **Supported Operations:** Addition (`+`), Subtraction (`-`), Multiplication (`*`), Division (`/`), Unary Negation (`-`), and Parentheses `()`.

## Project Structure

* `lexer.c / .h`: Tokenizes the raw input string.
* `parser.c / .h`: Recursive descent parser that builds the Abstract Syntax Tree.
* `ast.c / .h`: Defines the tree structure and an ASCII pretty-printer.
* `codegen.c / .h`: Compiles the AST into stack-based bytecode instructions.
* `vm.c / .h`: The virtual machine that executes the generated bytecode.
* `main.c`: The driver code that handles the CLI, REPL, file reading, and test suite.
* `Makefile`: Automates the build process.
